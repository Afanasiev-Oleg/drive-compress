- Никаких рефакторингов/переименований вне затронутых мест. Только минимальные изменения.
- MBperMin остаётся технической колонкой; COL.* и порядок столбцов не менять.
- Строки меню — точные (в т.ч. «Добрать длительность (Range — надёжный)» с ё).
- Range только для video/mp4 и video/quicktime. Это поведение не менять.
- Автопауза Range через RANGE_RESUME_DELAY_MS (и таймер, и текст тоста).
- Батч в GitHub: один repository_dispatch, event_type="drive_compress", ≤20 файлов.
- Использовать существующие константы/паттерны (новые — добавлять в config.gs).
- Не трогать форматирование/пустые строки вне контекста правок.

# Формат работы/ответа (строго)
1) PLAN (короткий): перечисли файлы и конкретные якоря/функции, которые изменишь (по 3–5 строк контекста).
2) ВНЕДРЕНИЕ ПРАВОК: вноси изменения НАПРЯМУЮ в файлы проекта средствами IDE AI (Apply changes).
   - Изменяй только заявленные участки. Без авто-реформатирования всего файла.
   - Один осмысленный коммит с понятным message. Никаких лишних diff’ов.
   - Если нужно — создай PR на main, без дополнительных правок вне задачи.
3) Краткое резюме: что изменено и как проверить (1–3 шага).
